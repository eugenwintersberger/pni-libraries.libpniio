Detailed Description

template<typename T, size_t N>
struct pni::container_utils< std::array< T, N > >

While std::vector and std::list are dynamic types whose size can be set at compile time, the size of std::array is fixed at compile time. Therefore, std::array requires some special treatment for array construction. This specialization of the container_utils template takes care about all the peculiarities of std::array.

inlinestatic

The syntax of how instances of std::array are constructed from initializer lists differes from that of std::vector and std::list. This function provides a common interface. If the number of elements in the initializer list does not match that of the std::array instance an exception will be thrown.

typedef std::array<float64,3> array_type;
auto v = container_utils<array_type>::create({1,2,3});

Initilize an instance of std::array from an iterator range. If the number of elemenst spanned by the iterator range does not match the number of elements in std::array an exception will be thrown. Otherwise, an instance of std::array initialized with the data refered to by the iterator range will be returned.

inlinestatic

Here we try to create a container of size n and initialize all elements with default_value. As n for std::array is already defined at compile time this method does not allocate new memory. If n does not match the size of std::array an exception will be thrown. Otherwise std::array will be initialized with default_value.

typedef std::array<float64,3> array_type;
auto vector = container_utils<array_type>::create(3,1.23);
